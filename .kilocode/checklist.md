# ✅ Kilo Code Commit Checklist

## Перед каждым коммитом
- [ ] Код компилируется без ошибок TypeScript
- [ ] ESLint/Prettier не показывают предупреждений
- [ ] Новые функции имеют тесты (минимум 1 кейс)
- [ ] Обработка ошибок добавлена (try/catch, error boundaries)
- [ ] Логи добавлены для критических путей
- [ ] Секреты не закоммичены (проверь .env)

## Для AI-функций
- [ ] Промпты логируются в dev-режиме
- [ ] Таймауты настроены (макс 60 сек для генерации)
- [ ] Fallback на вторую модель если первая упала
- [ ] Rate limiting реализован

## Для API-роутов
- [ ] Авторизация проверена
- [ ] Входные данные валидированы (Zod)
- [ ] SQL-инъекции исключены (parameterized queries)
- [ ] Response时间在 200ms для простых запросов

## Для UI-компонентов
- [ ] Loading-состояние есть
- [ ] Error-состояние есть
- [ ] Доступность (aria-labels, keyboard nav)
- [ ] Мобильная адаптация проверена

## Для базы данных
- [ ] Миграция создана и протестирована
- [ ] Индексы добавлены для частых запросов
- [ ] RLS (Row Level Security) настроен в Supabase
- [ ] Бэкап-стратегия документирована

## Для деплоя
- [ ] Environment variables задокументированы в .env.example
- [ ] CI/CD пайплайн проходит
- [ ] Health check endpoint работает
- [ ] Monitoring/alerting настроен
